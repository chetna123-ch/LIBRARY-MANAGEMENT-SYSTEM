# 📚 Advanced Library Management System (C++)

## 📌 Overview

The Advanced Library Management System is a console-based application developed in C++ that demonstrates the practical implementation of Object-Oriented Programming (OOP), Standard Template Library (STL), and core Data Structures & Algorithms.

Unlike basic CRUD-based systems, this project focuses on building an efficient and scalable solution by integrating optimized searching, sorting, and analytical techniques.

---

## 🚀 Key Features

- ➕ Add new books with multi-word title and author support  
- 📖 Display all books with status (Issued / Available)  
- 🔍 Search books using Binary Search (O(log n))  
- 🔎 Prefix-based search using string manipulation  
- 📤 Issue books and update status  
- 📥 Return books with validation  
- 🔃 Sort books using STL `sort()`  
- 🧠 Find the book with the longest title  
- 📊 Sliding Window analysis on book titles  
- 💾 Persistent data storage using file handling  

---

## 🧠 Concepts Used

### 🔹 Object-Oriented Programming (OOP)
- Class-based design (`Book`, `Library`)
- Encapsulation of data and operations
- Modular and maintainable code structure  

---

### 🔹 Standard Template Library (STL)
- `vector` for dynamic data storage  
- `sort()` for efficient sorting  

---

### 🔹 Data Structures & Algorithms

- **Binary Search** → Optimized book lookup (O(log n))  
- **Sorting Algorithms (STL)** → Efficient data organization  
- **String Manipulation** → Prefix-based search  
- **Sliding Window Technique** → Analytical feature for title comparison  

---

### 🔹 File Handling

- `ifstream` → Load data from file  
- `ofstream` → Save data persistently  
- Ensures data is not lost after program termination  

---

## 🛠️ Tech Stack

- **Language:** C++  
- **Paradigm:** Object-Oriented Programming  
- **Libraries:** STL (`vector`, `algorithm`)  
- **Storage:** File-based (`library.txt`)  

---

## ⚙️ How It Works

1. On startup, the system loads existing data from `library.txt`  
2. Users interact through a menu-driven interface  
3. All operations (add, issue, return) update both memory and file  
4. Searching is optimized using sorting + binary search  
5. Additional features provide analytical insights using DSA techniques
